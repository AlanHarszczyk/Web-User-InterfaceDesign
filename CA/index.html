HTTP(Hypertext Transfer Protocol)
Defines rules of requesting and transferring web resources which are HTML files,images and stylesheets.
HTTPS(Hypertext Transfer Protocol Secure)
Its a secure version of HTTP, it uses encription to protect data which is sent between user/client and the server. This ensures data is safe and confidential.
TCP/IP(Transmission Control Protocol/Internet Protocol)
TCP in charge of communication between application software such as browser and network software, breaks data into IP packs and sends and reasemmbles again. IP is routing the packs to their destination aka between computers, responsible for sending packets to reciever.
FTP(File Transfer Protocol)
Allows for transfer of local files to the web.

URL and DNS

URl(Uniform Resource Locator), they are web adresses that identify resources on the internet. They consist of the protocol (e.g. htttp), domain name(e.g www.example.com) and path (e.g./page.html). DNS(Domain Name System), translates human made domain names into IP adresses allowing for them to locate web servers by name rather than numerical IP adresses.

TLD(Top Level Domains)

Last segment of domain name located after dot. Examples include:

    .xyz
    .gov
    .int
    .biz
    .info

Cookies

Small pieces of data sent by clients browser. Used for authentication, tracking user behavoiur etc. They can remember site prefrences such geographic location to help with weather and traffic information giving you relevant local info.

Catching

Stores copies of frequently accesed web resources on client. Reduces load on web servers and improves page load time by serving cached context when available.
